# 키보드 단축키 구현 가이드

본 프로젝트의 사용자 인터페이스(UI)에서 키보드 단축키를 추가하고 관리하는 방법을 설명합니다.

## 1. 단축키 등록 방식

### 1.1 커맨드 기반 등록 (권장)
대부분의 일반적인 작업은 커맨드 시스템을 통해 단축키를 매핑합니다.

**설정 파일**: `[project-root]/src/config/shortcuts.ts` (또는 유사한 경로)

```typescript
export const defaultShortcuts = [
  { key: 's', ctrl: true, command: 'file:save' },
  { key: 'z', ctrl: true, command: 'edit:undo' },
  { key: 'y', ctrl: true, command: 'edit:redo' },
];
```

### 1.2 복합 단축키 (Chord Keys)
두 개 이상의 키를 순차적으로 입력하는 방식입니다. (예: `Ctrl+K` 이후 `B`)

```typescript
const chordMap = {
  'k+b': 'action:bookmark',
  'k+t': 'action:timestamp',
};
```

## 2. 단축키 처리 우선순위

시스템은 키 입력을 감지했을 때 다음 순서대로 처리합니다.

1. **시스템 예약키**: 브라우저나 OS에서 사용하는 기본 키 (F5, F12 등) 처리 여부 결정
2. **모달/다이얼로그**: 현재 열려 있는 팝업 창의 키 처리
3. **입력 필드**: 텍스트 입력창 내부에서의 포커스 및 입력 처리
4. **글로벌 단축키**: 설정된 매핑 테이블에 따른 커맨드 실행
5. **기본 동작**: 위 단계에서 처리되지 않은 키의 기본 동작 수행

## 3. 다국어 및 IME 대응

한글 등 조합형 문자(IME)를 사용하는 환경에서는 단축키 인식이 의도와 다를 수 있습니다.

- **한글 키 대응**: 단축키 등록 시 영어 키뿐만 아니라 해당 위치의 한글 키도 함께 등록하는 것을 고려합니다. (예: `Ctrl+L`과 `Ctrl+ㅣ`)
- **조합 상태 확인**: 문자가 조합 중(`isComposing`)일 때는 네비게이션 키(화살표 등) 처리를 보류해야 할 수 있습니다.

## 4. 새 단축키 추가 절차

1. **기능 구현**: 수행할 로직을 커맨드나 함수 형태로 구현합니다.
2. **매핑 등록**: 단축키 설정 파일에 키와 커맨드를 연결합니다.
3. **UI 표시**: 메뉴나 툴팁에 사용자에게 보일 단축키 라벨(예: `Ctrl+S`)을 추가합니다.
4. **충돌 확인**: 기존에 등록된 단축키와 겹치지 않는지 확인합니다.

---

> [!NOTE]
> 단축키는 숙련된 사용자의 생산성을 크게 향상시킵니다. 하지만 너무 많은 단축키는 학습 곡선을 높이므로, 직관적이고 표준적인(예: 저장=Ctrl+S) 매핑을 우선적으로 사용하세요.
