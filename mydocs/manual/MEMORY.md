# AI 공유 메모리 인덱스 (Shared Memory)

본 문서는 Antigravity가 프로젝트를 수행하며 반드시 기억하고 있어야 할 핵심 결정 사항, 트러블슈팅 경험, 그리고 작업지시자의 피드백을 기록한 '외부 메모리'입니다. Antigravity는 새로운 세션을 시작하거나 관련 작업을 수행하기 전에 이 목록을 반드시 확인해야 합니다.

## 1. 협업 및 프로세스 규칙
- **작업 종료 결정권**: Antigravity가 임의로 작업을 종료하거나 완료를 선언하지 않습니다. 최종 판단은 작업지시자가 합니다.
- **타스크 번호 관리**: 이슈 트래커(GitHub Issues 등)를 통해 번호를 자동 채번하며, 수동 번호 할당은 금지합니다.
- **절차 준수**: [이슈 → 브랜치 → 할 일 등록 → 계획서 승인 → 구현 → 보고] 순서를 절대 생략하지 않습니다.
- **문서 커밋 규칙**: 모든 보고서와 오늘 할 일 업데이트는 해당 타스크 브랜치에서 소스 코드와 함께 커밋합니다.

## 2. 설계 및 아키텍처 원칙
- **의존성 방향 준수**: 프로젝트의 핵심 모델은 외부 모듈에 의존하지 않아야 합니다.
- **결정성(Determinism) 유지**: 동일한 입력에 대해서는 항상 동일한 결과물(바이트 단위 일치)을 생성해야 합니다.
- **기술 부채 최소화**: 임시 우회 방식보다는 근본적인 해결책을 우선시합니다.

## 3. 검증 및 품질 기준
- **자기 검증의 한계 인지**: 시스템 내부 테스트를 통과했더라도, 실제 표준 소프트웨어나 환경에서의 최종 검증 게이트를 거쳐야 합니다.
- **트러블슈팅 사전 검색**: 유사한 작업 시작 전 `mydocs/troubleshootings/` 폴더를 검색하여 과거의 실수를 반복하지 않습니다.
- **렌더링 회귀 방지**: 시각적 변화가 있는 작업 후에는 반드시 스냅샷 테스트를 통해 의도치 않은 변경이 없는지 확인합니다.

## 4. 환경 및 도구 설정
- **출력 폴더 구조**: 생성되는 결과물은 용도별(재현용, 최종 결과용, 디버그용)로 지정된 서브폴더에 분리하여 저장합니다.
- **외부 리소스 경로**: 프로젝트 외부의 대용량 에셋이나 폰트 등의 경로 설정을 프로젝트 규칙에 맞게 관리합니다.

## 5. 주요 미결/보류 사항
- [특정 기능 A]: 현재 인프라 부족으로 구현 보류 중 (기기 확보 시 재개)
- [기술 검토 B]: 대안 기술 탐색 중 (차기 마일스톤 반영 예정)
