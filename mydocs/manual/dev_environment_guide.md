# 개발 환경 구축 가이드

본 프로젝트의 원활한 개발과 테스트를 위해 필요한 환경 설정 방법을 안내합니다. 모든 개발자는 (Antigravity 포함) 일관된 환경에서 작업해야 합니다.

## 1. 필수 도구 설치

프로젝트 성격에 따라 다음 도구들을 설치해야 합니다.

- **언어 및 런타임**: [예: Rust 1.75+, Node.js 18+]
- **버전 관리**: Git
- **가상화**: Docker (일관된 빌드 및 테스트 환경 보장)
- **에디터**: VS Code (권장 확장 도구 포함)

## 2. 프로젝트 초기화

저장소를 클론한 후 초기 설정을 진행합니다.

```bash
# 저장소 클론
git clone [repository-url]
cd [project-dir]

# 의존성 설치
[명령어 예: npm install 또는 cargo fetch]

# 환경 변수 설정
cp .env.example .env
# .env 파일을 열어 필요한 정보(API 키, 경로 등)를 수정하세요.
```

## 3. 빌드 및 실행

### 3.1 로컬 실행
개발 모드에서 시스템을 실행합니다.
```bash
[실행 명령어 예: npm run dev]
```

### 3.2 빌드
배포용 아티팩트를 생성합니다.
```bash
[빌드 명령어 예: npm run build]
```

## 4. 테스트 환경

### 4.1 단위 테스트
개별 모듈의 기능을 검증합니다.
```bash
[테스트 명령어 예: npm test 또는 cargo test]
```

### 4.2 컨테이너 기반 빌드 (권장)
OS 환경에 구애받지 않고 일관된 결과를 얻기 위해 Docker 사용을 권장합니다.
```bash
docker-compose up --build
```

## 5. 트러블슈팅

환경 구축 중 발생할 수 있는 일반적인 문제 해결 방법입니다.

- **의존성 충돌**: 캐시를 삭제하고 다시 설치하세요. (`npm cache clean --force` 등)
- **권한 오류**: 실행 권한을 확인하거나 관리자 권한이 필요한지 확인하세요.
- **환경 변수 미비**: `.env` 파일에 누락된 항목이 없는지 확인하세요.

---

> [!IMPORTANT]
> Antigravity는 새로운 환경에서 작업을 시작할 때 반드시 이 가이드에 따라 환경이 정상적으로 구축되었는지 테스트 명령어를 통해 확인해야 합니다.
