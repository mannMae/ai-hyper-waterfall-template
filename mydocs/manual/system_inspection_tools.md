# 시스템 분석 및 디버깅 도구 매뉴얼

본 프로젝트는 내부 데이터 구조와 실행 상태를 심층적으로 분석하기 위한 전용 CLI 도구를 제공합니다. 이 도구들은 Antigravity가 복잡한 버그를 스스로 진단하고, 작업지시자와 정확한 수치로 소통하는 데 사용됩니다.

## 1. 데이터 구조 분석 도구 (`dump`)

시스템의 내부 데이터 모델(Internal Representation)을 텍스트 형식으로 상세히 출력합니다.

### 1.1 기본 사용법
```bash
[tool-name] dump <대상_파일> [옵션]
```

### 1.2 주요 옵션
- `--section / -s`: 특정 섹션이나 모듈 데이터만 출력
- `--item / -i`: 특정 인덱스의 아이템 상세 정보 출력
- `--depth`: 재귀적 구조의 출력 깊이 조절

### 1.3 출력 항목 예시
- **메타데이터**: 소유자, 생성일, 수정 이력 등
- **구조 정보**: 계층적 객체 트리, 부모-자식 관계
- **속성 상세**: 각 객체의 세부 설정값 (좌표, 크기, 색상, 옵션 등)
- **참조 관계**: 고유 ID를 통한 객체 간 연결 상태

## 2. 상태 차이 분석 도구 (`diff`)

두 개의 데이터 모델이나 실행 결과를 비교하여 차이점을 식별합니다. 주로 회귀 테스트나 버전 간 비교에 사용됩니다.

### 2.1 기본 사용법
```bash
[tool-name] diff <기준_파일> <비교_파일> [옵션]
```

### 2.2 분석 시나리오
- **회귀 분석**: 코드 수정 전/후의 결과물 차이 감지
- **호환성 검증**: 원본 데이터와 시스템 처리 결과 간의 일치도 확인
- **결정성 테스트**: 동일한 입력에 대해 항상 같은 내부 상태를 생성하는지 확인

## 3. 디버그 오버레이 (Debug Overlay)

시각적 결과물(이미지, PDF, SVG 등) 위에 내부 식별자나 경계선을 겹쳐서 표시합니다.

### 3.1 활용 방법
```bash
[tool-name] export <대상> --debug-overlay
```

### 3.2 식별 정보
- **영역 표시**: 각 컴포넌트의 경계선(Bounding Box) 표시
- **식별자 라벨**: 각 객체의 고유 ID나 인덱스 표시
- **수치 표시**: 좌표(x, y) 및 크기(w, h) 정보 표시

## 4. 커뮤니케이션 가이드

Antigravity는 분석 도구의 출력을 바탕으로 작업지시자에게 보고해야 합니다.

- **나쁜 보고**: "데이터가 좀 이상하게 들어있습니다."
- **좋은 보고**: "`dump` 결과 `item_id=45`의 `padding_top` 값이 `50`으로 설정되어 있습니다. 이는 기대값 `10`보다 큽니다."

## 5. 도구 확장 방법

새로운 데이터 타입이나 로직이 추가될 경우, `dump`와 `diff` 명령어도 해당 정보를 지원하도록 확장해야 합니다.

- **Trait 구현**: 데이터 객체에 `Dumpable` 또는 `Diffable` 인터페이스 구현
- **포맷터 추가**: 가독성 높은 텍스트 또는 JSON 출력을 위한 전용 포맷터 등록

---

> [!TIP]
> Antigravity는 복잡한 문제를 해결하기 전에 반드시 `dump` 명령어로 현재 상태를 먼저 확인해야 합니다. 추측이 아닌 데이터에 기반한 디버깅이 Hyper-Waterfall의 원칙입니다.
