# 코드 품질 대시보드 운영 매뉴얼

본 프로젝트는 코드의 정량적인 품질 지표(Metrics)를 측정하고 시각화하기 위한 대시보드 시스템을 운영합니다.

## 1. 대시보드 개요

품질 대시보드는 코드의 복잡도, 크기, 테스트 통과율, 린트 경고 등을 한눈에 확인하여 기술 부채를 시각화하고 관리하는 것을 목적으로 합니다.

### 1.1 주요 구성 요소
- **메트릭 수집 스크립트**: 소스 코드를 분석하여 수치 데이터를 추출하는 도구
- **메트릭 데이터 (JSON)**: 수집된 정보를 저장하는 구조화된 데이터 파일
- **시각화 대시보드 (HTML)**: 데이터를 차트와 카드로 표시하는 웹 페이지

## 2. 지표 수집 및 대시보드 실행

### 2.1 메트릭 수집
```bash
# 메트릭 수집 스크립트 실행
[script-path]/metrics.sh
```

### 2.2 대시보드 열기
수집된 데이터를 바탕으로 생성된 HTML 파일을 브라우저에서 엽니다.
```bash
open output/dashboard.html
```

## 3. 주요 측정 항목 및 기준

대시보드에서 관리하는 핵심 지표와 권장되는 임계값입니다.

| 항목 | 측정 기준 | 목표/임계값 | 비고 |
|------|-----------|------------|------|
| **파일 길이 (LOC)** | 단일 파일의 총 줄 수 | 800~1,000줄 이하 | 초과 시 파일 분리 권고 |
| **인지 복잡도 (CC)** | 함수의 로직 복잡도 | 15 이하 (목표), 25 (경고) | 초과 시 리팩토링 대상 |
| **린트 경고 (Lint)** | 정적 분석 도구의 경고 수 | 0개 | 모든 경고 해결 필수 |
| **테스트 통과율** | 전체 테스트 중 성공 비율 | 100% | 실패 시 배포 불가 |
| **코드 커버리지** | 테스트가 실행된 코드 비율 | 70% 이상 | 중요 로직은 90% 이상 권장 |

## 4. 대시보드 차트 구성

- **파일 크기 분포**: 어떤 파일이 가장 비대해지고 있는지 확인
- **복잡도 Top N**: 가장 리팩토링이 시급한 함수 목록 표시
- **테스트 현황 (도넛 차트)**: 성공, 실패, 무시된 테스트 비중 시각화
- **히스토그램**: 전반적인 코드 크기 분포 및 변화 추이 파악

## 5. 관리 및 활용 방안

- **정기 점검**: 매 마일스톤 종료 시 대시보드를 확인하여 품질 저하 여부를 체크합니다.
- **PR 승인 기준**: 새로운 코드가 유입될 때 복잡도가 급격히 상승하거나 테스트 통과율이 낮아지는지 검토합니다.
- **임계값 조정**: 프로젝트의 성격과 규모에 맞춰 `thresholds` 설정을 주기적으로 업데이트합니다.

---

> [!TIP]
> "측정할 수 없는 것은 관리할 수 없다." 대시보드에 나타난 붉은색 지표들은 시스템이 보내는 경고 신호입니다. 기능을 추가하기 전에 먼저 품질 지표를 개선하는 시간을 가지세요.
