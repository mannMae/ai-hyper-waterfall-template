# AI 페어프로그래밍 가이드

## 개요

이 문서는 원본 프로젝트에서 **Claude**와 사람이 협업하며 축적한 페어프로그래밍 패턴을 바탕으로, 본 프로젝트의 에이전트인 **Antigravity(with Gemini)** 환경에 맞춰 템플릿화한 가이드입니다.

Hyper-Waterfall 방법론을 바탕으로, 작업지시자의 도메인 지식과 Antigravity의 구현력이 결합되어 고도의 집중력과 속도로 고품질의 결과물을 생산하는 것을 목표로 합니다.

## 역할 분담

### 작업지시자 (사람)

- **방향 설정**: 무엇을 해야 하는지, 왜 해야 하는지를 결정
- **도메인 지식**: 프로젝트의 핵심 비즈니스 로직, 사용자 요구사항, 시각적 품질 판단
- **품질 기준**: "우회 방식은 허용하지 않습니다", "업계 표준을 준수해야 합니다"
- **즉각적 피드백**: 실행 결과 캡처, 로그 출력, "이 기능은 이렇게 동작해야 합니다" 같은 구체적 교정
- **작업 시간 관리**: 작업의 시작과 종료를 결정

### Antigravity (AI 페어 프로그래머)

- **구현**: 코드 작성, 빌드, 테스트 실행
- **분석**: 코드 구조 탐색, 버그 원인 추적, 로직 계산
- **설계**: API 설계, 자료구조 설계, 테스트 프레임워크 제안
- **문서화**: 매뉴얼, 커밋 메시지, 계획서 및 보고서 작성
- **검증**: 단위 테스트, 통합 테스트, 회귀(Regression) 확인

## 핵심 원칙

### 1. 구체적인 피드백으로 즉시 교정한다

```
작업지시자: [실행 화면/로그 첨부] "이 부분의 출력이 예상과 다릅니다"
   → Antigravity: 원인 파악 → 코드 수정 → 재실행 결과 보고
작업지시자: [확인] "이제 맞습니다" 또는 "여전히 10% 정도 오차가 있습니다"
   → Antigravity: 추가 보정 → 재확인
```

정량적이고 시각적인 증거를 바탕으로 소통합니다. 작업지시자가 구체적인 데이터나 캡처를 제공하면 Antigravity가 이를 분석하여 정확한 해결책을 도출합니다.

### 2. 우회하지 않고 정면 돌파한다

작업지시자가 임시방편이나 우회 방식을 거부하면, 근본적인 해결책을 찾아야 합니다. 기술 부채를 남기지 않는 것이 Hyper-Waterfall의 핵심입니다.

- **우회**: "일단 하드코딩으로 해결하겠습니다" → **거부됨**
- **정면 돌파**: 논리적 구조 재설계 → 아키텍처 개선 → 근본적인 문제 해결

### 3. 검증 가능한 구조를 만든다

수동 확인에 의존하면 회귀 오류(Regression)를 잡을 수 없습니다.

```
수동 확인 → "정상적으로 보입니다"
   ↓ 발전
자동 검증 → 자동화된 테스트 시나리오 → 상태/출력값 측정 → 규칙 기반 판정
```

**발전 과정**:
1. 실행 결과 덤프/캡처로 눈으로 확인
2. 자동화된 단위 테스트 추가
3. 시나리오 기반 통합 테스트 구축
4. CI/CD 파이프라인을 통한 자동 검증

### 4. 작업지시자가 방향을 잡고 Antigravity가 실행한다

Antigravity가 스스로 작업 범위를 정하거나 임의로 종료를 결정하지 않습니다.

- 작업지시자: "이번 타스크에 이 범위까지 포함시켜 주세요" → Antigravity: 범위 확장
- 작업지시자: "이것은 별도 이슈로 등록하고 다음으로 넘어가세요" → Antigravity: 이슈 분리
- 작업지시자: "좋은 것 같습니다. 작업 종료하세요" → Antigravity: 최종 보고 후 종료

### 5. 간과한 것을 발견하면 즉시 반영한다

작업지시자의 한 마디가 설계 방향을 근본적으로 바꿀 수 있습니다. Antigravity는 대화 중 나오는 새로운 제약사항이나 통찰을 즉시 아키텍처에 반영해야 합니다.

## 효과적인 커뮤니케이션 패턴

### 구체적 증거 기반 대화

| 비효율적 | 효과적 |
|---------|--------|
| "동작이 이상합니다" | [로그/캡처] "A 함수 호출 후 B 상태가 되어야 하는데 C 상태로 남습니다" |
| "성능이 느립니다" | "10,000개 데이터 처리 시 5초가 걸립니다. 목표는 1초 이내입니다" |
| "디자인이 틀립니다" | [참고 이미지] "이 버튼은 배경색과 대비가 더 커야 합니다" |

### 점진적 정밀화

1. **1차**: "A 기능을 구현해야 합니다" (방향 설정)
2. **2차**: "이런 조건들을 만족해야 합니다" (규칙 정의)
3. **3차**: "엣지 케이스(예: 데이터 없음)를 고려해 주세요" (예외 처리)
4. **4차**: "출력 형식을 JSON에서 YAML로 변경해 주세요" (상세 조정)

## 디버깅 워크플로우

Antigravity는 다음 도구들을 활용하여 스스로 문제를 진단할 수 있어야 합니다.

1. **상태 덤프**: 내부 변수나 객체의 상태를 텍스트로 출력하여 분석
2. **로깅**: 주요 실행 흐름을 추적할 수 있는 정교한 로그 활용
3. **단위 테스트**: 문제 상황을 재현하는 최소 단위의 테스트 코드 작성
4. **차이 분석(Diff)**: 이전 정상 상태와 현재 오류 상태의 출력값 비교

## 프로젝트 구조가 AI 페어프로그래밍에 미치는 영향

AI 친화적인 프로젝트 구조는 생산성을 극대화합니다.

- **CLI 도구 체인**: AI가 즉시 실행하여 결과를 확인할 수 있는 명령줄 도구
- **빠른 테스트**: 몇 초 안에 수백 개의 테스트를 수행할 수 있는 환경
- **모듈화**: 기능별로 명확히 분리되어 AI가 특정 부분에 집중할 수 있는 구조
- **명확한 문서화**: `ANTIGRAVITY.md`와 같은 규칙 문서

## 체크리스트: Antigravity와 효과적으로 작업하기

### 작업 시작 전

- [ ] 이슈 등록 및 타스크 번호 할당
- [ ] 작업 브랜치 생성
- [ ] 재현 가능한 샘플 또는 요구사항 명세 준비

### 작업 중

- [ ] 구체적인 증거(로그, 캡처) 기반으로 피드백 제공
- [ ] "맞음/틀림"을 명확히 전달
- [ ] 우회가 아닌 근본 해결을 요구
- [ ] 중간 결과(테스트 통과 여부 등)를 자주 확인

### 작업 완료

- [ ] 전체 테스트 통과 확인
- [ ] 문서화(매뉴얼, 보고서) 완료 여부 체크
- [ ] 최종 승인 후 브랜치 병합 및 이슈 종료

## 결론

AI 페어프로그래밍의 핵심은 **역할 분담의 명확성**입니다. 사람은 방향·판단·피드백을, Antigravity는 구현·분석·검증을 담당합니다. 이 분담이 명확할수록 작업 속도와 품질이 동시에 높아집니다.
